# Issue #100: apiFetch가 백엔드 응답 래퍼를 unwrap하지 않아 런타임 TypeError 발생

> https://github.com/GreenBrain-Inha/FE_GreenBrain/issues/100
> Labels: bug, frontend | Milestone: 없음
> 생성일: 2026-05-25

## 목표
`lib/api.ts`의 `apiFetch`에서 백엔드 응답 래퍼 `{ success, message, data }` 중 `data` 필드를 자동 unwrap하여, 모든 API 호출부가 정상 동작하도록 수정.

## 관련 파일
- `lib/api.ts` — apiFetch 함수 수정 (unwrap 로직 추가)
- `components/ChatSidebar.tsx` — 기존 방어 코드 정리
- `app/challenges/feed/page.tsx` — 기존 방어 코드 정리
- `contexts/AppContext.tsx` — apiFetch 사용
- `app/chat/page.tsx` — apiFetch 사용
- `app/profile/page.tsx` — apiFetch 사용
- `components/ChallengeModal.tsx` — apiFetch 사용
- `app/login/page.tsx`, `app/signup/page.tsx`, `app/onboarding/page.tsx` — apiFetch 사용

## 작업 분해
- [x] 이슈 분석 및 API 응답 래퍼 구조 확인
- [ ] `lib/api.ts` — `response.json()` 후 `.data` unwrap 로직 추가
- [ ] `components/ChatSidebar.tsx` — 기존 `Array.isArray` 방어 코드 제거 (unwrap로 불필요)
- [ ] `app/challenges/feed/page.tsx` — 기존 `Array.isArray` 방어 코드 제거
- [ ] 전체 34개 apiFetch 호출부 검증 — unwrap 후에도 타입 일치 확인

## 완료 기준
- `apiFetch`가 `{ success, message, data }`에서 `data`만 반환
- `/chat` 페이지에서 `sessions is not iterable` 에러 미발생
- `/challenges/feed` 페이지에서 `Cannot read properties of undefined` 에러 미발생
- 기존 방어 코드(`Array.isArray`) 제거 후에도 정상 동작
