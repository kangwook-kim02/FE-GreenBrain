## 목표
로그인·회원가입·로그아웃 API를 실제 백엔드와 연동한다.

## 관련 파일
- `app/login/page.tsx` — `POST /api/auth/login` 연동, 성공 시 `/chat`, 실패 시 인라인 에러
- `app/signup/page.tsx` — `POST /api/auth/signup` 연동, 성공 시 `/onboarding`, 실패 시 인라인 에러
- `contexts/AppContext.tsx` — `logout()` 함수 추가 (API 호출 + Context 초기화 + `/login` 이동)

## 작업 분해
- [ ] login/page.tsx: `POST /api/auth/login` 연동 (성공→/chat, 401→인라인, 429→횟수초과 인라인)
- [ ] signup/page.tsx: `POST /api/auth/signup` 연동 (성공→/onboarding, 400→중복 이메일 인라인, 422→형식 오류)
- [ ] AppContext: `logout()` 추가 — `POST /api/auth/logout` + `setUser(null)` + `setTokens` 초기화 + `/login` 이동

## 완료 기준
- 로그인 성공 → `/chat` 이동
- 로그인 실패(401) → "이메일 또는 비밀번호가 올바르지 않습니다" 인라인
- 로그인 실패(429) → "로그인 시도 횟수를 초과했습니다" 인라인
- 회원가입 성공 → `/onboarding` 이동
- 회원가입 실패(400) → "이미 사용 중인 이메일입니다" 인라인
- 로그아웃 → Context 초기화 + `/login` 이동
- 쿠키(JWT)는 백엔드 Set-Cookie 처리, localStorage 저장 없음
