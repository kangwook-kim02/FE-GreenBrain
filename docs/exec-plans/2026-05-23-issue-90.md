# Issue #90: 페이지 이동 시 ChatSidebar 언마운트·리마운트로 인한 깜빡임

> https://github.com/GreenBrain-Inha/FE_GreenBrain/issues/90
> Labels: bug, frontend | Milestone: -
> 생성일: 2026-05-23

## 목표

페이지 이동(chat → profile → feed) 시 ChatSidebar가 매번 언마운트·리마운트되어 깜빡이는 현상 제거.
Next.js Route Group 레이아웃으로 SidebarLayout을 한 번만 마운트하고, 콘텐츠 영역만 교체.

## 원인

각 페이지가 `<SidebarLayout>`을 직접 렌더링 → 페이지 이동 시 이전 페이지 언마운트 + 새 페이지 마운트 = SidebarLayout과 ChatSidebar가 매번 새로 생성됨.
`sidebarOpen = false → useEffect로 true`로 바뀌는 사이클이 반복되어 데스크톱에서도 순간적으로 사라졌다 나타남.

## 관련 파일

- `components/SidebarLayout.tsx` — render prop 패턴 → ReactNode 패턴으로 변경
- `contexts/SidebarContext.tsx` — 신규: toggleButton을 Context로 공유
- `app/(main)/layout.tsx` — 신규: Route Group 공유 레이아웃
- `app/chat/page.tsx` → `app/(main)/chat/page.tsx`
- `app/profile/page.tsx` → `app/(main)/profile/page.tsx`
- `app/challenges/feed/page.tsx` → `app/(main)/challenges/feed/page.tsx`

## 작업 분해

- [x] `contexts/SidebarContext.tsx` 생성 — `useSidebar()` 훅 export
- [x] `components/SidebarLayout.tsx` 수정 — `children: (toggleButton) => ReactNode` → `children: ReactNode` + Context.Provider
- [x] `app/(main)/layout.tsx` 생성 — SidebarLayout 단일 인스턴스
- [x] 세 페이지를 route group으로 이동 + SidebarLayout 래핑 제거 + `useSidebar()` 사용
- [x] 원본 page 파일 삭제

## 완료 기준

- 페이지 이동 시 사이드바 깜빡임 없음
- 세 페이지 (/chat, /profile, /challenges/feed) URL 변경 없음
- 로그인·회원가입·온보딩 페이지에는 사이드바 미노출
