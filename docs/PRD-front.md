# PRD: GreenBrain 프론트엔드 (MVP)

> 작성일: 2026-05-04 | 최종 수정: 2026-05-12 | 팀: GreenBrain
원본 PRD(`doc/PRD.md`)를 기반으로 프론트엔드 구현 범위만 분리한 문서입니다.
디자인 프로토타입(`DesignPrototype_GreenBrain`)의 최종 상태를 반영합니다.

## 1. 역할 경계

프론트엔드는 **UI 렌더링, 사용자 인터랙션, 클라이언트 유효성 검사, API 호출**을 담당한다.
비즈니스 로직, 데이터 저장, 보안 정책은 백엔드 책임이며 프론트엔드는 백엔드 응답을 신뢰한다.

| 책임 | 프론트엔드 | 백엔드 |
| --- | --- | --- |
| 화면 렌더링 | ✅ |  |
| 클라이언트 유효성 검사 (형식) | ✅ |  |
| API 호출 및 응답 처리 | ✅ |  |
| 로딩 / 에러 / 빈 상태 표시 | ✅ |  |
| JWT 토큰 저장 (HttpOnly 쿠키) |  | ✅ |
| 탄소 계산 (ecologits) |  | ✅ |
| AI 응답 생성 (GPT-4o mini) |  | ✅ |
| 토큰 차감 / 회복 계산 |  | ✅ |
| 챌린지 자동 생성 |  | ✅ |
| 좋아요 보상 지급 |  | ✅ |
| 어뷰징 방지 |  | ✅ |
| 파일 저장 / 리사이즈 |  | ✅ |
| 일일 토큰 초기화 |  | ✅ |

## 4. 레이아웃 구조

### 4.1 데스크탑 (sm: 640px 이상)

```
+--사이드바(w-64)--+--------메인 콘텐츠--------+
| 🌱 GreenBrain   |  헤더 (토글 버튼 + 타이틀)  |
| [새 채팅]        |  페이지 본문               |
| [챌린지 피드]    |                            |
| [프로필]         |                            |
| ──────────────  |                            |
| 오늘            |                            |
|  · 채팅 1       |                            |
|  · 채팅 2       |                            |
| 어제            |                            |
|  · 채팅 3       |                            |
+-----------------+----------------------------+
```

- 사이드바는 토글 버튼으로 열기/닫기 가능 (`w-64` ↔ `w-0` 너비 트랜지션)
- 사이드바 기본 상태: 열림
- 사이드바 배경: `bg-gray-900` (다크)
- 활성 페이지에 해당하는 사이드바 항목: `bg-gray-700` + 좌측 초록 테두리(`border-l-2 border-green-400`) 강조

### 4.2 모바일 (sm 미만)

- 사이드바 미표시
- 화면 하단 고정 탭 바 (`fixed bottom-0`): 채팅 / 피드 / 프로필
- 활성 탭: 초록색 아이콘 + 상단 인디케이터 선

## 5. 화면별 UI 요구사항

### 5.1 로그인 (`/login`)

**UI 구성**

- 로고 + 서비스 설명
- 이메일 입력 필드
- 비밀번호 입력 필드
- 로그인 버튼
- 회원가입 링크

**클라이언트 유효성 검사**

- 이메일 형식 확인
- 비밀번호 미입력 방지

**API 연동**

```
POST /api/auth/login
Body: { email, password }
성공 → /chat 이동 (쿠키는 백엔드가 Set-Cookie로 설정)
실패 → 에러 메시지 표시 ("이메일 또는 비밀번호가 올바르지 않습니다")
```

**상태 처리**

- 로딩: 버튼 비활성화 + 스피너
- 에러: 폼 하단 인라인 에러 메시지

---

### 5.2 회원가입 (`/signup`)

**UI 구성**

- 이메일 입력 필드
- 비밀번호 입력 필드
- 비밀번호 확인 입력 필드
- 가입하기 버튼
- 로그인 링크

**클라이언트 유효성 검사**

- 이메일 형식 확인
- 비밀번호: 최소 8자, 대소문자 + 숫자 혼합 (정규식)
- 비밀번호 일치 여부

**API 연동**

```
POST /api/auth/signup
Body: { email, password }
성공 → /onboarding 이동
실패 → 에러 메시지 표시 ("이미 사용 중인 이메일입니다" 등)
```

**상태 처리**

- 로딩: 버튼 비활성화 + 스피너
- 에러: 폼 하단 인라인 에러 메시지

### 5.3 온보딩 (`/onboarding`)

**UI 구성**

- 단계 표시 (1/3, 2/3, 3/3) + 프로그레스바
- 각 단계별 질문 + 선택지 카드 (아이콘 + 라벨)
- 이전 / 다음 버튼

**질문 구성**

| 단계 | 질문 | 선택지 |
| --- | --- | --- |
| 1 | 주로 사용하는 교통수단 | 대중교통, 자가용, 자전거, 도보 |
| 2 | 식단 유형 | 비건, 채식 위주, 균형 식단, 육식 위주 |
| 3 | 주거 형태 | 아파트, 단독주택, 원룸/오피스텔, 셰어하우스 |

**인터랙션**

- 선택된 카드: 초록 테두리 + 배경 + 체크 아이콘 표시
- 선택 전 다음 버튼 비활성화
- 3단계 완료 후 저장 API 호출

**API 연동**

```
POST /api/user/onboarding
Body: { transportation, diet, housing }
성공 → /chat 이동
실패 → 에러 토스트 표시
```

---

### 5.4 채팅 (`/chat`)

**UI 구성**

- 좌측 사이드바 (데스크탑): 새 채팅 버튼, 피드/프로필 링크, 채팅 히스토리 목록
- 헤더: 사이드바 토글 버튼 + 로고 + 탄소 토큰 바 + 챌린지 참여 버튼
- 메시지 영역: 첫 진입 시 환영 화면 → 채팅 시작 후 메시지 목록으로 전환
- 하단 입력창 + 전송 버튼

**환영 화면 (첫 진입 시 — 메시지 없을 때)**

- 🌱 아이콘 + `{닉네임}님, 다시 오셨네요` 대형 텍스트
- 서브텍스트: `오늘도 함께 탄소를 줄여봐요`
- 입력창 placeholder: `오늘 어떤 도움을 드릴까요?`

**채팅 진행 중**

- 사용자 메시지: 오른쪽 초록 말풍선
- AI 메시지: 왼쪽 흰색 말풍선
- AI 메시지 하단: 탄소 배출 정보 카드 (gCO₂eq 수치 + 비유 텍스트)
- 입력창 placeholder: `메시지를 입력하세요.`

**챌린지 참여 버튼**

- 위치: 토큰 프로그레스바 우측 하단
- 텍스트: `챌린지에 참여하여 토큰 회복하기`
- 클릭 시 `ChallengeModal` 팝업 표시

**토큰 프로그레스바 색상**

| 잔여 비율 | 색상 |
| --- | --- |
| 50% 초과 | 초록 (green-500) |
| 20% ~ 50% | 앰버 (amber-500) |
| 20% 미만 | 빨강 (red-500) |

**API 연동**

```
// 페이지 진입 시 현재 토큰 조회
GET /api/user/tokens
응답: { remaining: number, max: 150 }

// 채팅 히스토리 목록
GET /api/chat/history
응답: { groups: [{ label: "오늘"|"어제"|"이전 7일", items: [{ id, title }] }] }

// 메시지 전송
POST /api/chat/message
Body: { message: string }
응답: { reply: string, carbon_cost: number, remaining_tokens: number }
```

**상태 처리**

- 메시지 전송 중: 말풍선 로딩 애니메이션 (... 점 세 개)
- ecologits 실패 시: 탄소 정보 카드에 "탄소 계산 불가" 표시 (토큰 차감 없음 — 백엔드 판단)
- 토큰 소진 시: 입력창 비활성화 → 토큰 소진 안내 화면 전환

**토큰 소진 화면** (토큰 0 상태)

- 경고 아이콘 + "오늘의 탄소 토큰을 모두 사용했습니다" 메시지
- "챌린지 시작하기" 버튼 → `ChallengeModal` 오픈

### 5.5 챌린지 모달 (`ChallengeModal`)

> `/challenges` 독립 페이지를 운영하지 않고, `/chat` 헤더의 버튼으로 모달을 진입한다.
모바일: 하단 시트(bottom sheet) / 데스크탑: 중앙 다이얼로그
> 

**단계 1 — 챌린지 소개**

- 챌린지 카테고리 뱃지 + 아이콘 + 제목 + 설명
- 보상 안내 (커뮤니티 좋아요 3개당 +20 토큰)
- "챌린지 수락하기" 버튼

**단계 2 — 사진 업로드**

- 챌린지 정보 요약
- 사진 업로드 영역 (파일 선택 버튼 + 미리보기)
- "인증 사진 업로드" 버튼

**예외 상태**

- 일일 3개 상한 도달: "오늘의 챌린지를 모두 완료했습니다" 안내

**파일 클라이언트 유효성 검사**

- 허용 포맷: JPEG, PNG, WebP
- 최대 크기: 10MB
- 위반 시 인라인 에러 메시지 표시 (`alert()` 사용 금지)

**API 연동**

```
// 현재 챌린지 조회 (모달 오픈 시)
GET /api/challenges/current
응답: { challenge: Challenge | null, daily_count: number }

// 챌린지 수락
POST /api/challenges/{id}/accept
응답: { challenge: Challenge }

// 사진 업로드
POST /api/challenges/{id}/verify
Body: FormData { photo: File }
응답: { remaining_tokens: number }
성공 → 토큰 업데이트 후 모달 닫기
```

**상태 처리**

- 조회 중: 모달 내 스피너
- 업로드 중: 버튼 비활성화 + "업로드 중..." 텍스트

### 5.6 인증 피드 (`/challenges/feed`)

**UI 구성**

- 커뮤니티 배너 (좋아요 안내)
- 피드 카드 목록 (최신순)
    - 업로더 아바타 + 닉네임 + 업로드 시각
    - 챌린지 제목 + 아이콘
    - 인증 사진 (정방형)
    - 좋아요 버튼 + 좋아요 수
    - 본인 게시물에만 삭제 버튼 (휴지통 아이콘, 우측 상단)
- 빈 상태: "아직 인증된 챌린지가 없습니다" + 챌린지 참여 버튼

**좋아요 버튼 상태**

| 조건 | 버튼 상태 |
| --- | --- |
| 기본 | 활성화, 비선택 (회색) |
| 이미 좋아요한 사진 | 비활성화, 초록 강조 |
| 본인이 업로드한 사진 | 비활성화, 회색 |

**삭제 버튼**

- 본인 게시물(`isOwner: true`)에만 표시
- 클릭 시 확인 다이얼로그 표시 후 삭제 요청

**API 연동**

```
// 피드 목록 조회
GET /api/feed?page=1
응답: { items: FeedItem[], has_next: boolean }
// FeedItem: { id, userId, username, challengeTitle, challengeIcon,
//             imageUrl, likes, likedByMe, isOwner, timestamp }

// 좋아요
POST /api/feed/{item_id}/like
응답: { likes: number, liked_by_me: boolean }
실패(중복/본인/신규계정) → 에러 메시지 표시

// 게시물 삭제 (본인만)
DELETE /api/feed/{item_id}
성공 → 목록에서 즉시 제거
```

**상태 처리**

- 조회 중: 스켈레톤 카드 3개
- 좋아요 요청 중: Optimistic UI (즉시 +1 반영 → 실패 시 롤백)
- 빈 상태: 안내 메시지 + 챌린지 참여 버튼

### 5.7 프로필 (`/profile`)

**UI 구성**

- 프로필 카드
    - 아바타 (이니셜 또는 업로드 사진) + 카메라 아이콘 버튼 (사진 변경)
    - 닉네임 + 인라인 편집 버튼 (연필 아이콘) → 저장 버튼
    - 이메일 표시 (수정 불가)
- 생활습관 프로필 카드 (교통수단 / 식단 / 주거 형태)
    - 현재 선택값 표시 + "변경" 버튼
    - "변경" 클릭 시 해당 섹션 옵션 카드 펼침 (아코디언)
    - 옵션 선택 즉시 저장 + 섹션 닫힘
- 로그아웃 버튼

**API 연동**

```
// 닉네임 수정
PATCH /api/user/profile
Body: { nickname: string }

// 프로필 사진 업로드
POST /api/user/avatar
Body: FormData { avatar: File }
응답: { avatar_url: string }

// 생활습관 수정
PATCH /api/user/onboarding
Body: { transportation?, diet?, housing? }

// 로그아웃
POST /api/auth/logout
성공 → 전역 Context 초기화 + /login 이동
```

**상태 처리**

- 닉네임 저장 중: 저장 버튼 비활성화
- 생활습관 저장 중: 옵션 카드 클릭 비활성화
- 에러: 해당 섹션 인라인 에러 메시지

## 7. 전역 상태 관리

React Context로 관리할 전역 상태:

```tsx
interface AppState {
  user: {
    id: string;
    email: string;
    nickname: string;
    onboarding_complete: boolean;
  } | null;
  tokens: {
    remaining: number;   // 백엔드 응답 기준
    max: number;         // 150 고정
  };
}
```

- **토큰**: API 응답마다 `remaining_tokens`로 동기화. 클라이언트에서 자체 계산하지 않음.
- **사용자 정보**: 로그인 성공 시 설정, 로그아웃 시 초기화.
- **닉네임**: 채팅 환영 화면, 프로필 페이지 모두 Context의 `user.nickname`을 읽어 표시.

## 8. API 에러 공통 처리

| HTTP 상태 | 처리 |
| --- | --- |
| 401 | 로그인 페이지로 리다이렉트 |
| 403 | 인라인 에러 메시지 표시 |
| 422 | 필드별 유효성 에러 메시지 표시 |
| 500 | "일시적인 오류가 발생했습니다" 토스트 |
| 네트워크 오류 | "인터넷 연결을 확인해주세요" 토스트 |
