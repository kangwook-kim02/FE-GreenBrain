# PRD: GreenBrain (MVP)

> AI 챗봇 탄소 책임 서비스
작성일: 2026-04-27 | 팀: GreenBrain
> 

---

## 1. 프로젝트 개요

### 기술 스택

프론트엔드: Next.js, TypeScript, shadcn, Tailwind CSS

백엔드: Supabase, PostgreSQL, FastAPI, OpenAI api, ecologits

### 1.1 문제 정의

사람들은 AI 챗봇을 쓸 때마다 탄소가 발생한다는 사실을 알고 있지만, **얼마나 발생하는지 체감하지 못하기 때문에** 행동으로 이어지지 않는다.

- AI 도구를 하루 수십 번 쓰는 사람도 탄소 발생량을 직관적으로 인식하지 못한다.
- 기존 환경 앱은 탄소를 '기록'하거나 '정보'로 보여줄 뿐, **소비 제한이나 행동 유도로 이어지지 않는다.**
- "환경을 위해 뭔가 해야 하는데"라는 막연한 의식은 있지만, 일상의 작은 행동으로 연결되지 않는다.

### 1.2 솔루션

GreenBrain은 AI 챗봇 사용량을 탄소 토큰으로 환산하여 보여주고, **토큰을 소진하면 실생활 탄소 절감 챌린지를 수행하고 인증 사진과 커뮤니티 좋아요로 채팅 토큰을 회복하게 만드는** 탄소 책임 서비스다.

- 탄소 소비를 숫자로 가시화해 체감을 유도
- 토큰 소진 → 챌린지 수행 → 사진 인증 → 좋아요 → 토큰 회복의 루프로 행동 변화 유도
- MVP는 ecologits 기반 탄소 계산과 커뮤니티 좋아요 중심으로 빠르게 구현

### 1.3 타겟 사용자

AI 도구를 일상적으로 사용하는 20-35세.
환경에 관심은 있으나 행동으로 이어지지 않는 사람들.
"나도 탄소 줄여야 하는데"라고 생각하면서 LLM을 하루 수십 번 쓰는 사람.

### 1.4 핵심 차별점

| 항목 | 기존 환경 앱 | "지속가능한 AI" 캠페인 | GreenBrain |
| --- | --- | --- | --- |
| 핵심 | 탄소 기록/정보 | 인식 제고 | **탄소 책임 행동 유도** |
| AI 연계 | X | X | O (AI 사용 자체가 소비 단위) |
| 행동 연결 | X | X | O (챌린지 수행 + 사진/좋아요 → 토큰 회복) |
| 커뮤니티 | 없음 | 없음 | O (사진 인증 + 좋아요 검증) |
| 개인화 | 없음 | 없음 | O (생활 습관 프로필 기반 챌린지) |

### 1.5 MVP 완료 기준

MVP는 사용자가 아래 핵심 루프를 end-to-end로 수행할 수 있으면 완료로 본다.

- 회원가입,  로그인, 온보딩을 완료할 수 있다.
- 채팅을 보내면 GPT-4o mini 응답과 메시지별 탄소 배출량을 확인할 수 있다.
- 일일 탄소 토큰이 소진되면 채팅 입력이 차단되고 챌린지 화면으로 이동한다.
- 토큰 소진 시 사용자는 한 번에 하나의 자동 생성 챌린지를 받을 수 있다.
- 사용자는 챌린지 인증 사진을 업로드하면 토큰 20개를 즉시 회복할 수 있다.
- 인증 사진이 다른 사용자 좋아요 3개를 받을 때마다 토큰 20개를 추가 회복할 수 있다.
- 하루에 생성할 수 있는 챌린지는 최대 3개이며, 좋아요로 얻을 수 있는 토큰은 하루 최대 50개다.

## 2. 기능 요구사항

### 2.1 탄소 토큰 기반 채팅

| ID | 기능 | 설명 |
| --- | --- | --- |
| F-C01 | 채팅 | GPT-4o mini 기반 AI 채팅 |
| F-C02 | 탄소 배출량 계산 | 메시지마다 ecologits로 실제 gCO₂eq 계산 |
| F-C03 | 토큰 표시 | 채팅 UI 상단에 남은 탄소 토큰 실시간 표시 (숫자 + 프로그레스바) |
| F-C04 | 토큰 색상 변화 | 잔여 토큰에 따라 그린 → 앰버 → 레드로 색상 전환 |
| F-C05 | 토큰 소진 처리 | 토큰 소진 시 입력창 비활성화 → 챌린지 화면으로 전환 |
| F-C06 | 일일 토큰 초기화 | 사용자 관점에서 매일 KST 기준 150 탄소 토큰(=150 gCO₂eq)으로 초기화 |
| F-C07 | 일일 좋아요 보상 상한 | 좋아요로 회복할 수 있는 토큰은 하루 최대 50개로 제한 |

**토큰 정책**

- 탄소 토큰은 gCO₂eq 단위로 표시하며, 1 탄소 토큰은 1 gCO₂eq로 본다.
- 일일 기본 지급량은 150 토큰이다.
- 챌린지 인증 사진 업로드 시 해당 챌린지에서 1회만 20 토큰을 즉시 회복한다.
- 인증 사진이 다른 사용자 좋아요 3개를 받을 때마다 20 토큰을 추가 회복한다.
- 좋아요 보상은 누적 좋아요 수 3, 6, 9개 등 3개 단위에 도달할 때마다 1회 지급한다.
- 좋아요로 회복할 수 있는 토큰은 하루 최대 50개다.
- 좋아요 보상으로 지급 가능한 잔여 상한이 20보다 작으면 남은 상한만큼만 지급한다.
- 좋아요 일일 상한에 도달한 뒤 발생한 좋아요 보상은 토큰을 지급하지 않으며, MVP에서는 다음 날로 이월하지 않는다.
- 사진 업로드 보상은 하루 챌린지 3개 제한에 따라 하루 최대 60 토큰까지 받을 수 있다.

**엣지 케이스**

- ecologits 호출 실패 시: 요청을 차단하지 않고, 해당 메시지에 "탄소 계산 불가" 표시. 해당 메시지는 토큰을 차감하지 않는다.
- 차감 결과 토큰이 0 이하가 되면 소진 상태로 처리하고, 사용자에게 표시되는 잔여 토큰은 0으로 고정한다.

### 2.2 탄소 절감 챌린지 자동 생성

| ID | 기능 | 설명 |
| --- | --- | --- |
| F-CH01 | 챌린지 자동 생성 | 토큰 소진 시 생활 습관 프로필 + 이력 기반으로 챌린지 1개 자동 생성 |
| F-CH02 | 중복 방지 | 진행 중인 챌린지가 있으면 새 챌린지를 만들지 않고 기존 챌린지 반환 |
| F-CH03 | 난이도 점진 상승 | 누적 완료 횟수에 따라 챌린지 난이도 자동 조정 |
| F-CH04 | 챌린지 카테고리 | 교통(대중교통, 도보), 식단(채식, 배달 금지), 에너지(에어컨 절약 등) |
| F-CH05 | 챌린지 수락 | 사용자가 챌린지를 수락하면 수행 중 상태로 전환 |
| F-CH06 | 일일 챌린지 상한 | 사용자당 하루 최대 3개까지만 챌린지 생성 |

**챌린지 정책**

- 토큰을 다 쓰면 챌린지는 한 번에 하나씩만 생성된다.
- 하루에 생성할 수 있는 챌린지는 최대 3개다.
- 사진 업로드는 챌린지 수행 인증 상태를 의미한다.
- 사진 업로드가 완료되면 해당 챌린지는 완료 상태가 되며, 이후 토큰을 다시 소진하면 새 챌린지를 받을 수 있다.
- 챌린지 생성은 토큰 소진 상태에서만 가능하다.
- 진행 중인 챌린지가 있거나 일일 챌린지 3개 상한에 도달하면 토큰이 다시 소진되어도 새 챌린지를 추가 생성하지 않는다.

### 2.3 사진 인증 & 커뮤니티 좋아요

| ID | 기능 | 설명 |
| --- | --- | --- |
| F-V01 | 사진 업로드 | 챌린지 수행 인증 사진 업로드 |
| F-V02 | 인증 피드 | 다른 사용자의 인증 사진 목록 표시 |
| F-V03 | 좋아요 버튼 | 다른 사용자가 엄지 아이콘으로 좋아요 |
| F-V04 | 업로드 보상 | 인증 사진 업로드 시 업로더 토큰 +20 즉시 회복 |
| F-V05 | 좋아요 보상 | 인증 사진 좋아요 3개당 업로더 토큰 +20 회복 |

**피드 정책**

- 인증 피드는 사진이 업로드된 챌린지만 최신순으로 표시한다.
- 본인이 업로드한 사진은 좋아요 버튼을 비활성화한다.
- 이미 좋아요한 사진은 `liked_by_me` 상태로 표시하고 재좋아요를 막는다.
- 좋아요 보상은 이미 지급된 3개 단위 구간에 대해 중복 지급하지 않는다.

**어뷰징 방지**

- 동일 사용자가 같은 사진 중복 좋아요 불가
- 본인 사진에 본인 좋아요 불가
- 계정 생성 후 24시간 이내 신규 계정은 좋아요 버튼 비활성화

**파일 처리**

- 허용 포맷: JPEG, PNG, WebP / 최대 10MB
- 서버에서 리사이즈(최대 1280px) 후 저장
- MVP: 로컬 파일시스템 / 프로덕션: S3 호환 스토리지

### 2.4 인증 / 온보딩

| ID | 기능 | 설명 |
| --- | --- | --- |
| F-A01 | 회원가입 | 이메일 + 비밀번호 |
| F-A03 | 로그인 / 로그아웃 | JWT 기반 |
| F-A04 | 온보딩 설문 | 가입 직후 생활 습관 3문항 수집 |

**보안 요구사항**

- 비밀번호: 최소 8자, 대소문자 + 숫자 혼합
- JWT: HttpOnly 쿠키에 저장, 만료 7일
- Rate limiting: 로그인 시도 5회 초과 시 15분 잠금

### 2.5 비기능 요구사항

| 항목 | 요구사항 |
| --- | --- |
| 플랫폼 | MVP는 반응형 웹으로 제공한다. 모바일 앱은 제공하지 않는다. |
| 브라우저 | 최신 Chrome, Safari, Edge에서 핵심 플로우가 동작해야 한다. |
| 상태 처리 | 주요 화면은 로딩, 빈 상태, 에러 상태를 제공해야 한다. |
| API 경계 | LLM 및 ecologits 호출은 FastAPI 백엔드에서만 수행한다. |
| 파일 보관 | MVP 사진 파일은 로컬 파일시스템에 저장하고, 프로덕션에서는 S3 호환 스토리지로 전환한다. |
| 개인정보 | 이메일, 비밀번호 해시, 세션 토큰, 업로드 사진은 서버에서만 관리하고 클라이언트에 민감 정보를 노출하지 않는다. |

## 4. 사용자 플로우

### 4.1 회원가입 플로우

```
이메일 + 비밀번호 입력
  ↓
회원가입 완료
  ↓
온보딩 3문항 (주 교통수단 / 식단 유형 / 주거 형태)
  ↓
채팅 화면 진입
```

### 4.2 메인 플로우 (채팅 → 토큰 소진 → 챌린지 인증)

```
채팅 화면 (토큰 잔여량 표시)
  ↓
AI 대화 → ecologits 탄소 계산 → 토큰 차감
  ↓
[토큰 소진]
  ↓
채팅 입력 비활성화 → ChallengeModal 팝업 진입 (별도 페이지 이동 없음)
  ↓
챌린지 자동 생성 (생활 습관 프로필 + 이력 반영)
  ↓
사용자 챌린지 수락 → 수행 → 사진 업로드
  ↓
업로드 즉시 +20 탄소 토큰 회복
  ↓
피드에 노출 → 다른 사용자가 좋아요 클릭
  ↓
좋아요 3개당 +20 탄소 토큰 회복 (하루 최대 50 토큰)
  ↓
채팅 화면 복귀
```

### 4.3 인증 피드 플로우

```
피드 화면
  ↓
다른 사용자의 챌린지 인증 사진 목록 확인
  ↓
엄지 버튼으로 좋아요 (본인 사진 / 중복 / 신규 계정 불가)
  ↓
좋아요 누적 3개 단위 도달 시 업로더 토큰 +20 회복 (하루 최대 50 토큰)
```

## 5. 화면 목록

| 화면 | URL | 설명 |
| --- | --- | --- |
| 로그인 | `/login` | 이메일 + 비밀번호 로그인 |
| 회원가입 | `/signup` | 이메일 가입 |
| 온보딩 | `/onboarding` | 생활 습관 프로필 3문항 |
| 채팅 | `/chat` | 메인 화면. 탄소 토큰 + AI 채팅. 챌린지 모달 진입점 |
| 인증 피드 | `/challenges/feed` | 다른 사용자 인증 사진 목록 + 좋아요 버튼 |
| 프로필 | `/profile` | 닉네임·프로필 사진·생활 습관 수정, 로그아웃 |

> **참고**: `/challenges` 는 독립 페이지로 운영하지 않는다.
> 챌린지는 `/chat` 화면 내 ChallengeModal 팝업으로 진입한다.
