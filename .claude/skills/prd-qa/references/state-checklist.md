# 페이지별 상태 처리 체크리스트

> fe-qa가 각 화면의 상태 처리 완성도를 검증할 때 이 파일을 기준으로 사용한다.

---

## Login (`/login`)

| 상태 | 구현 기준 |
|------|----------|
| **로딩** | 제출 버튼 `disabled` + 텍스트 "로그인 중..." |
| **서버 에러** | `bg-red-50 border border-red-200 text-red-700` 인라인 박스 |
| **필드 에러** | 이메일/비밀번호 필드 아래 `text-red-500 text-sm` (react-hook-form) |
| **빈 상태** | 해당 없음 |

---

## Signup (`/signup`)

| 상태 | 구현 기준 |
|------|----------|

| **서버 에러** | 인라인 박스 (이미 사용 중인 이메일 등) |
| **필드 에러** | 이메일/비밀번호 필드 아래 인라인 |
| **빈 상태** | 해당 없음 |

---

## Onboarding (`/onboarding`)

| 상태 | 구현 기준 |
|------|----------|
| **로딩** | "완료" 버튼 `disabled` + 텍스트 "저장 중..." |
| **서버 에러** | 토스트 또는 인라인 박스 |
| **선택 전** | "다음" / "완료" 버튼 `disabled` (항목 미선택 시) |
| **빈 상태** | 해당 없음 |

## Chat (`/chat`)

| 상태 | 구현 기준 |
|------|----------|
| **메시지 전송 중** | 3-dot 애니메이션 (`animate-bounce`) 말풍선 표시 |
| **전송 버튼 로딩** | 전송 버튼 `disabled` (전송 중 재전송 방지) |
| **ecologits 실패** | `carbon_cost: null` → "탄소 계산 불가" 표시 (수치 미표시) |
| **토큰 소진** | `remaining_tokens === 0` → 입력창 비활성화 + 챌린지 유도 메시지 |
| **채팅 히스토리 로딩** | 사이드바 히스토리 로딩 스켈레톤 또는 로딩 인디케이터 |
| **히스토리 빈 상태** | 해당 없음 (빈 사이드바 허용) |
| **에러** | 전송 실패 시 인라인 에러 또는 토스트 |

## ChallengeModal (챌린지 모달)

| 상태 | 구현 기준 |
|------|----------|
| **현재 챌린지 로딩** | 스켈레톤 또는 로딩 스피너 |
| **일일 한도 도달** | `daily_count >= 3` → 새 챌린지 생성 불가 안내 |
| **업로드 진행 중** | "인증 중..." + 버튼 `disabled` |
| **파일 유효성 에러** | 인라인 에러 텍스트 (`text-red-500 text-xs`) — `alert()` 금지 |
| **업로드 실패** | 서버 에러 인라인 표시 |
| **챌린지 없음** | `challenge: null` → 새 챌린지 제안 UI |

## ChallengeFeed (`/challenges/feed`)

| 상태 | 구현 기준 |
|------|----------|
| **초기 로딩** | 스켈레톤 카드 3개 (`bg-gray-100 rounded-xl h-48 animate-pulse`) |
| **빈 상태** | "아직 인증된 챌린지가 없습니다." + "첫 챌린지 시작하기" CTA 버튼 |
| **좋아요 처리** | Optimistic UI — 서버 응답 전에 UI 즉시 반영, 실패 시 롤백 |
| **좋아요 에러** | 인라인 에러 (중복/본인/신규계정) — 토스트 또는 인라인 |
| **삭제 처리** | Optimistic UI — 서버 응답 전 목록에서 즉시 제거 (204 기준) |
| **페이지 로딩 에러** | 에러 메시지 + 재시도 버튼 |

## Profile (`/profile`)

| 상태 | 구현 기준 |
|------|----------|
| **닉네임 저장 중** | 저장 버튼 `disabled` + "저장 중..." |
| **닉네임 저장 에러** | 필드 아래 인라인 에러 |
| **아바타 업로드 중** | 업로드 버튼 `disabled` + 진행 표시 |
| **아바타 업로드 에러** | 인라인 에러 텍스트 |
| **생활습관 저장 중** | 해당 섹션 저장 버튼 `disabled` |
| **생활습관 저장 에러** | 섹션 아래 인라인 에러 |
| **로그아웃 처리 중** | 버튼 `disabled` |
| **빈 상태** | 해당 없음 |

---

## 공통 규칙

1. **`alert()` 사용 절대 금지** — 모든 에러는 인라인으로 처리
2. **네트워크 에러** — `catch` 블록에서 "인터넷 연결을 확인해주세요." 표시
3. **401 응답** — `/login` 리다이렉트
4. **500 응답** — "일시적인 오류가 발생했습니다" 토스트 또는 인라인
5. **로딩 중 중복 요청 방지** — `isLoading` / `isSubmitting` true일 때 버튼 `disabled`
